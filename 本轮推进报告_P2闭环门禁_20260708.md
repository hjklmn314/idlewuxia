## 38 条 result token P2 是什么

1. 时间/定时标记缺执行器：17 条
   - 影响：任务重复、冷却、拜访、领取类逻辑可能不准。
   - 下一步：在 runtime player state 增加 `playerTimeMarkers` / `playerTimedMarkers`，执行“玩家时间标记设置”“玩家定时标记设置”，并让条件判断能读取这些标记。

2. 技能经验缺执行器：9 条
   - 影响：第一章互动奖励给基础拳脚、轻功、内功、刀法、招架、读书识字、一气功等技能经验时，现在还没有真正写入技能成长。
   - 下一步：增加数据驱动的 `skillExp` / `skillProgression` 状态，按 result token 的 `Arg2=skillId`、`Arg3=expDelta` 写入，并补技能经验/等级门槛判断。

3. 季节/活动委托：5 条
   - 影响：百年、百年委托等活动型内容不是第一章主线闭环核心，但不能静默当作已实现。
   - 下一步：保留为 `defer_with_guard`，入口必须标注非首章范围，不能在首章验收里冒充完成。

4. 政绩/系统进度副作用：4 条
   - 影响：改变政绩类 result token 可能属于更外层账号/系统进度。
   - 下一步：继续追 Lua/配置字段归属；若属于当前可见流程，就实现 `meritLedger` 或同等数据结构，否则把对应分支隐藏出首章验收范围。

5. 副本故事对白：3 条
   - 影响：唐门故事类文本现在没有统一走对白/日志反馈链。
   - 下一步：把“副本故事” result 的 `Arg2` 文本接入已有 narrative feedback/dialogue pipeline，保留分号分段。

## 8 条 flow parity P2 是什么

1. 导航桥接策略：7 条
   - 包括任务列表返回、章节卡返回、地图返回、NPC 交互返回、离开等。
   - 这些不是竞品代码中已证实的功能命令，而是为了浏览器/app 原型可验收增加的项目策略。
   - 下一步：允许保留，但必须数据驱动、标记为 design proposal，且真实浏览器验收必须证明不会跳到错误界面，例如不能再从挂机返回到“华山”这种错误状态。

2. 隐藏项目桥接节点：1 条
   - `NODE_FB01_SETTLEMENT_LOOP` 属于项目内部桥接/展示节点。
   - 下一步：只允许隐藏存在，不允许作为玩家可见竞品流程节点出现。
