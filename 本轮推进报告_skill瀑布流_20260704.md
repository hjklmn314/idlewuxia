# 本轮推进报告：Skill 瀑布流与首局真实浏览器验收

日期：2026-07-04

## 使用的 Skill

- `wuxia-project-automation`：组织端口守门、自动回归、浏览器验收和报告闭环。
- `wuxia-system-design`：保证首局流程继续按 `StateId / ActionId / ServerCommand / ViewModel` 数据驱动，不把节点、任务、奖励写死在界面代码里。
- `wuxia-game-planning`：区分竞品证据、我方映射和仍需验证的推断。
- `wuxia-ux-ui-design`：修复玩家点击任务/节点后缺少明确反馈的问题，检查玩家是否能看懂当前状态和下一步。
- `browser:control-in-app-browser`：用真实浏览器点击链路验收，不再用离线截图替代。

## 本轮已修

1. 清掉入口旧 Nova 污染：
   - `index.html` 改为武侠首局专用 shell。
   - 不再保留旧 `gameCanvas`、`GALAXY`、`TURRET`、`Offline`、`Welcome Back` 静态 DOM。
   - `src/main.js` 对 `mode=nova` 做保护，不让旧原型顶层 canvas 逻辑污染武侠入口。

2. 改稳定端口：
   - `tools/dev-server.mjs` 默认端口改为 `5187`。
   - 当前验收入口：`http://127.0.0.1:5187/`。
   - `127.0.0.1:5177` 不再作为武侠主原型入口，避免被旧 prototype / Nova Lite 服务抢占。

3. 增加端口守门：
   - `tools/assert-wuxia-dev-server.mjs`
   - 命令：`npm run wuxia:assert:dev-server`
   - 守门目标：确认当前服务返回 `Idle Wuxia Prototype`、`wuxiaShell`，且没有旧 Nova 文本信号。

4. 第一章节点变成可执行入口：
   - `config/wuxia_first_session_flow.json` 给主线/挑战节点补了 `primaryAction`。
   - 已补节点：
     - `NODE_FB01_OUTER_GATE`
     - `NODE_FB01_FRONT_YARD`
     - `NODE_FB01_MAIN_HALL`
   - `src/wuxiaFirstSessionFlow.js` 的 `selectChapterNode()` 会把 `primaryAction` 返回给 UI。
   - `src/main.js` 的节点详情会显示“进入某节点”按钮，点击后通过配置里的 `ActionId` 进入下一状态。

5. 补状态可审计信号：
   - 每次渲染都会写入：
     - `body.dataset.runtime = wuxia`
     - `body.dataset.wuxiaState`
     - `body.dataset.wuxiaScreen`
     - `body.dataset.wuxiaMode`
   - 点击地图节点后会写入 `body.dataset.wuxiaSelectedNode`。

6. 新增首局竞品证据映射：
   - 新工具：`tools/build-wuxia-first-session-evidence-map.mjs`
   - 新命令：`npm run wuxia:evidence:first-session`
   - 新输出：
     - `outputs/wuxia_first_session_evidence_map/first_session_competitor_evidence_map.csv`
     - `outputs/wuxia_first_session_evidence_map/first_session_competitor_evidence_map.json`
     - `outputs/wuxia_first_session_evidence_map/summary.json`
   - 作用：把首局 `source_file / state / action / chapter1_node / gate / reward / screen` 全部落成可追源映射表，后续逐条对照竞品录屏、Lua 和配置。
