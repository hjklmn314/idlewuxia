# 本轮推进报告：T01-04 Android 设备验收

## 结果

T01-04 的功能施工与自动设备门禁已完成。目标设备为雷电 Android 9
`emulator-5554`，型号 `V1938T`。自动验收 7/7 通过，设备日志阻断命中为
0，最终界面焦点属于 `com.idlewuxia.app.debug/com.idlewuxia.app.MainActivity`。

本轮首次设备冷启动暴露了两个此前桌面/Node 门禁无法发现的 P0 兼容问题：

1. Android WebView 不提供 `structuredClone`，首局配置加载直接失败。
2. 修复第一处后，WebView 又因 `Array.prototype.at` 缺失而失败；同一发布
   闭包中的 `String.prototype.replaceAll` 也被提前纳入兼容门禁。

兼容修复不是散落 polyfill。`src/dataClone.js` 成为唯一数据克隆 seam，
首局运行时与战斗预览均通过该模块；活动发布源码禁止直接使用上述旧
WebView 不兼容调用。模块测试在删除全局 `structuredClone` 后仍验证深拷贝
隔离，真实 Android 9 冷启动也从错误面板恢复为首局界面。

## 存档施工

原项目没有持久化接口。设备实测中，HOME 与锁屏可以保持内存状态，但
Android Back 销毁 Activity 后会回到开场，强停重启同样丢失出身选择。

本轮新增：

- `config/runtime_persistence_contract.json`：版本、存储键、事件上限和
  生命周期保存策略的数据源。
- `src/runtimePersistence.js`：恢复、自动保存、版本拒绝、损坏存档拒绝、
  容量限制和存储异常隔离的深模块。
- `createFirstSessionRuntime().exportSaveState()`：只导出玩家、状态机、旗标、
  任务、地图可变状态、待决战斗与受限事件，不复制章节静态配置。
- Android Back/强停重启后恢复 `STATE_FS_001_ORIGIN_RESULT`、`武学世家` 和
  事件计数的模块级与设备级双重验证。

## 自动设备门禁

`npm run device:validate -- --adb <path> --serial <serial>` 现在自动执行：

1. 安装指定 APK 并清空应用数据。
2. 冷启动、错误面板断言、9:16 视口断言、首屏截图。
3. 按实时 WebView 尺寸和 DPR 换算 ADB 点击，完成玩家出身选择。
4. HOME 前后台、真实锁屏/解锁、Android Back 重开、强停重开。
5. 终态焦点、存档状态、截图、完整 logcat 和阻断模式检查。
6. 输出带 APK 字节数和 SHA-256 的机器报告。

本轮保留的本地证据位于
`outputs/android_device_acceptance_20260717/`，包括自动报告、冷启动/终态
截图、完整日志、逐步骤状态文本和连续录像。该目录与 APK 均被 Git 忽略。

## 门禁边界

T01-04 设备功能门禁已经通过。正式 R1 关闭仍以任务提交后的干净修订重新
构建为准：`android:audit:wuxia` 必须 `formalAudit.ready=true`，然后使用该
正式 APK 再跑一次 `device:validate`，两份报告的 APK SHA-256 必须一致。

独立业务债务保持不变：`wuxia:check:fast` 中首局模拟器的既有
`rejected=2 / expectedRejected=1` 不属于本轮 Android 兼容、存档或设备链
修改，不在 T01-04 中伪装关闭。
